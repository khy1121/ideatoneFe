# 기획 수신 시 FE가 바로 물어볼 질문 목록

기획이 나오면 프론트엔드 입장에서 빠르게 확인해야 할 질문들을 정리했습니다. 기획자/디자이너/백엔드와 협의할 때 체크리스트로 활용하세요.

---

## 화면 관련 질문

- 필요한 페이지는 몇 개인가?
- 사용자 흐름(시나리오)은 어떻게 되는가? (예: 메인 → 입력 → 결과 → 저장)
- 메인 → 입력 → 결과 → 저장 흐름이 맞는가?
- 로그인/회원 기능이 필요한가? (익명 사용 가능한가?)
- 모바일 대응(반응형) 필수인가? 별도의 모바일 디자인이 있는가?
- 각 페이지에서 필요한 액션(버튼)과 우선순위는 무엇인가?
- 에러/empty 상태 화면(예: 결과 없음, 네트워크 오류) 디자인은 어떻게 할 것인가?

## 데이터 관련 질문

- 각 화면에 필요한 데이터 필드는 무엇인가?
- 사용자가 입력하는 값은 무엇인가? (텍스트, 파일, 선택지 등)
- 입력값의 유효성(밸리데이션) 규칙은 무엇인가? (최대 길이, 필수 여부 등)
- AI로부터 어떤 결과 필드를 받아야 하는가? (예: title, summary, checklist, documents, score 등)
- 분석 결과의 데이터 타입과 중첩 구조는 어떻게 되는가? (배열/객체/문자열)
- 저장해야 하는 데이터는 무엇인가? (전체 응답 vs 일부 요약)
- 결과 항목에 ID가 부여되는가? (결과 조회용 키)
- 민감한 데이터(개인정보 등)를 처리하는가? 저장/암호화 정책은?

## API 관련 질문

- 분석 요청 API가 필요한가? (POST `/api/analyze` 등)
- 결과 조회 API가 필요한가? (GET `/api/result/{id}` 등)
- 저장 API가 필요한가? (POST `/api/save` 등)
- 로그인/회원 관련 API가 필요한가? (로그인, 회원가입, 토큰 발급)
- 마이페이지 관련 API가 필요한가? (사용자 저장 목록 조회, 삭제)
- 요청/응답의 성공/에러 공통 포맷은 무엇인가? (예: {status, data, message})
- 응답 필드 이름과 경로를 명확히 해줄 수 있는가? (`data.summary` vs `data.result.summaryText` 같은 불일치 방지)
- 인증 방식(세션, JWT, 쿠키 등)은 무엇인가?
- 파일 업로드(예: 첨부파일)가 필요하면 엔드포인트와 전송 방식(multipart/form-data)을 정의해달라.

## 디자인 관련 질문

- Figma(또는 다른 디자인 파일)가 제공되는가? 공유 가능한 링크가 있는가?
- 색상, 폰트, 버튼 스타일 등 디자인 토큰이 정리되어 있는가?
- 공통 컴포넌트 기준(버튼, 입력, 모달, 카드 등)이 정해져 있는가?
- 반응형(브레이크포인트) 기준이 정해져 있는가?
- 아이콘/이미지 자산은 누가, 어떤 형식으로 제공하나?
- 접근성(aria) 요구사항이나 키보드 네비게이션 고려가 필요한가?

---

이 파일을 참고해 기획이 도착하면 우선순위가 높은 질문부터 기획자/백엔드/디자이너에게 확인하세요. 질문에 대한 답이 나오면 `API_CONTRACT.md`, `FE_STRUCTURE.md`, `FE_STYLE_GUIDE.md` 등의 문서를 업데이트하면 됩니다.

## FE 작업용 즉시 질문 체크리스트 (우선순위)

아래는 FE 작업을 시작할 때 가장 먼저 확인해야 할 핵심 질문들입니다. 기획 회의 직후 이 목록을 기반으로 빠르게 확인하세요.

- 화면/흐름: 필요한 페이지와 기본 사용자 흐름은 무엇인가요? (`/`, `/analyze`, `/result`, `/mypage` 등으로 충분한가?)
- 인증: 로그인/회원 기능이 필요한가요? 필요하면 인증 방식(JWT/세션)을 알려주세요.
- 핵심 입력값: 사용자가 입력하는 값(텍스트/파일/옵션)과 각 필드의 유효성 규칙은 무엇인가요?
- 필수 API: 분석 요청(POST `/api/analyze`), 결과 조회(GET `/api/result/{id}`), 저장(POST `/api/save`) 중 어떤 것이 반드시 필요한가요?
- 응답 구조: 분석 결과에서 반드시 필요한 필드(예: `id`, `title`, `summary`, `checklist`)를 확정해 주세요.
- 저장/조회: 결과를 저장하고 조회해야 하나요? 저장 시 어떤 필드를 보존해야 하나요?
- 디자인 자산: Figma 링크와 색상/폰트/버튼 스타일 등 디자인 토큰을 공유해 주세요.
- 우선 작업 범위: MVP로 먼저 만들어야 할 페이지/기능은 무엇인가요? (우선순위 지정)

이 체크리스트로 빠른 합의가 이루어지면 FE 작업 착수 후 초기 컴포넌트, API 호출 구조, 페이지 템플릿을 확정하고 개발을 진행할 수 있습니다.
