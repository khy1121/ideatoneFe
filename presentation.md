# 나독 기획 정리

## 프로젝트 한 줄 소개

틀에 박힌 독서 경험에서 벗어나, AI 캐릭터와 토론하며 나답게 독서하는 습관을 돕는 독서기록 앱.

## 문제의식

- 기존 독서기록앱은 책을 저장하고 기록하는 기능에만 머무는 경우가 많다.
- 책이 아니라 일기처럼, 사용자의 감정과 하루를 함께 기록하는 흐름이 필요하다.
- SNS 비교 피로와 트렌드 번아웃 속에서 사용자가 자기만의 속도로 읽고 생각할 수 있는 경험이 필요하다.

## 서비스 차별점

- AI 토론 기능을 통해 더 심도 있는 독서 경험을 제공한다.
- 라이프스타일에 스며드는 AI 캐릭터가 사용자의 독서 습관을 나답게 유지하도록 돕는다.
- 책 내용만이 아니라 사용자의 성향과 맥락을 함께 반영해 대화를 이끈다.

## 주요 기능

### Section 1. 로그인 / 회원가입

- 프로토타입 단계에서는 더미값 기반의 인증 흐름을 사용한다.
- 이메일, 비밀번호, 비밀번호 확인, 닉네임 단계로 이어지는 기본 가입 흐름을 구성한다.

### Section 2. 홈화면 / 메인

- 메인 화면은 서재, 책꽂이 형태로 저장된 책을 확인하는 구조다.
- 사용자가 책을 하나 선택하면 책 정보, 가독이챗, 메모 화면으로 이어진다.
- 메인 진입점은 와이어프레임을 기준으로 후속 업데이트 가능하도록 유지한다.

### Section 3. AI 캐릭터 생성 / AI 챗봇(토론) 페이지

- "오늘 너의 하루는 어때" 같은 입력을 바탕으로 AI 책속 주인공 캐릭터를 생성한다.
- 생성된 캐릭터는 독서길잡이 역할을 하며, 사용자가 독서를 더 재미있게 이어가도록 돕는다.
- AI 채팅봇(가독이)을 통해 책 내용 기반 토론과 분석 경험을 제공한다.

### Section 4. 마이페이지

- 사용자가 저장한 결과와 독서 이력을 확인하는 공간이다.
- 추후 저장, 조회, 관리 기능을 확장할 수 있도록 구조를 잡는다.

### Section 5. 기획 PPT

- 서비스 문제 정의, 핵심 가치, 주요 화면 흐름, 기능 우선순위를 정리한 기획 자료다.
- 화면 구현의 기준은 이 PPT와 와이어프레임을 기반으로 한다.

## 서비스 흐름 메모

- 입력 → 캐릭터 생성 → 토론 → 저장의 흐름을 중심으로 확장한다.
- 책 선택 후 책 정보, 가독이챗, 메모로 이어지는 상세 흐름을 유지한다.

## 데이터 소스 메모

- 책 데이터는 db 또는 알라딘 연동을 고려한다.

## 작업 기준

- `FE_ROUTES.md` 는 초안으로 보고, 개발이 진행되면 계속 업데이트한다.
- `MainPage` 같은 임시 화면은 와이어프레임 기준으로 교체 또는 재정의할 수 있다.
- 최종 구현 기준은 와이어프레임과 기획 PPT다.
