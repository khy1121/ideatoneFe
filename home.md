1️⃣ 홈화면

(1) 메인 홈

<aside>

1. 앱에 들어가자마자 나오는 화면
2. **사용자가 저장한 책들**이 한줄에 3개 씩 선반 위에 올려져 있다. 책이 없더라도 선반 모양은 존재. 무한 스크롤 가능. (계속 끝없이 스크롤 되는게 가능할까요 ??) 
3. 돋보기를 클릭하면 검색 화면으로 이동한다. 
4. **책을 클릭하면 책정보 페이지로 넘어간다. (이 페이지 우측 상단엔 수정 및 삭제 버튼이 뜬다.**
</aside>

2️⃣ 수정 및 삭제 페이지

(2) 수정 페이지

<aside>

1. 우측 상단에 있는 수정을 누를 시에 출력되는 기능이다.
2. 뒷 화면은 블러처리 된 채 밑에서 네모박스가 올라온다.
3. 독서상태에 따라 독서 기간 형태도 달라진다.
    1. 다 읽은 책: 독서기간이 시작일~종료일 모두 입력 가능
    2. 읽고 있는 책: 시작일 칸만 존재
    3. 찜한 책: 독서 기간 아예 존재X
    4. 이러한 독서 상태는 마이페이지에서 확인 가능하다. 
4. 수정하기 버튼을 클릭하면 수정이 완료되며 해당 박스는 나가지고 다시 이전의 책정보 페이지로 돌아간다. 
</aside>

(3) 삭제 페이지

<aside>

1. 우측 상단에 있는 삭제를 누를 시에 출력되는 기능이다.
2. 뒷 화면은 블러처리된 채, 사용자에게 정말로 삭제를 묻는 팝업창이 뜬다. ‘삭제 시 복구할 수 없습니다. 정말 삭제할까요?’
    1. 취소를 누르면 팝업창은 사라지며 다시 책정보 페이지로 돌아간다.
    2. 확인을 누르면, 해당 책정보 페이지에서 나가지며 서재 홈화면으로 돌아간다. 해당 책이 없어진 걸 확인할 수 있다. 
</aside>

3️⃣ 검색

(4) 검색 화면

<aside>

1. 사용자가 새로운 책을 검색하여 서재에 저장하기 위해 존재하는 화면. 
2. 검색창을 클릭하면 타자판이 올라며 글자를 입력할 수 있다. 초성 입력 가능 
3. 완료 버튼을 누를 때 까지 예시 리스트는 안올라옴. 화면 변경X (따로 완료버튼X. 키보드에 있는 버튼) 
</aside>

(5) 검색 리스트 화면

<aside>

1. 책 리스트들이 나열된다. 정렬순은 무조건 관련도순. 
2. 한 칸 당 왼쪽엔 책 표지 사진이 있고, 사진 옆엔 책 이름이 왼쪽 정렬로 써져있다. 그 밑엔 출판사와 책이 나온 년도가 작은 글씨로 써져있다. 
3. 해당 칸을 클릭하면 책 정보 소개 페이지로 넘어간다. 
</aside>

4️⃣ 책 정보 소개 페이지

(4) 저장

<aside>

1. 우측 상단엔 ‘저장’ 버튼이 있다.
    1. ‘저장’ 버튼을 눌러도 서재에 추가되기만 하지, 따로 사용자의 화면이 이동하진 않는다.
    2. 저장 버튼을 누르면(저장이 완료되면) 저장 버튼은 → 수정 및 삭제 버튼으로 바뀐다. (이 기능은 아래쪽에서 설명)
    3. **백버튼을 누르면 다시 한단계씩 이전 페이지로 돌아간다.  (바로 메인홈화면x)** 
2. ‘저장’ 버튼을 누르면 위에서 수정 및 삭제 페이지에서 떴던 네모 박스 기능과 일치. 
3. **다만 하단의 수정하기 버튼 →  저장하기 버튼으로 바뀐다.**
</aside>

(5) 책 정보 소개 

<aside>

1. 가운데 정렬로, 책 제목과 책 표지 이미지, 저자 가 나타나있다. 
    1. 이는 수정 및 클릭 불가
2. 사용자의 책 분류, (마이페이지에서 확인 가능), 독서 기간 이 나타나있다. 이는 저장 후 사용자가 수정 가능.
3. 그 밑엔 책정보, 가독이챗, 메모 기능이 있다. 
4. **책정보**를 클릭하면 책 소개, 출판사, ISBN, 페이지 정보가 뜬다. 스크롤을 통해 내려볼 수 있다. (DB: )
    1. 책 정보는 수정 불가.
5. **가독이챗**을 클릭 시에 ‘가독이챗 하러 가기’버튼이 있다. 이를 클릭하면 가독이쳇 메인 화면으로 이동한다.

</aside>
